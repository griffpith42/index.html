01 The Problem

Analysts answer
data questions.
Not anymore.

At every credit fund and bond analytics firm, a structural bottleneck exists: analysts lose 1 to 2 hours per day retrieving data instead of analyzing it. A client asks a question. An analyst opens Excel. Twenty minutes pass.

DetQ eliminates that bottleneck. It sits as a query layer on top of your existing database. Analysts type natural language. DetQ returns structured, accurate results in seconds, with no human intervention required.

The key architectural insight: AI models should extract intent, not make decisions. Every filter, ranking, and output rule is enforced deterministically in code after the AI runs. The result is a system that cannot hallucinate your data.

The Time Problem

1–2 hrs

Lost per analyst per day to manual data retrieval in a standard 8am–5pm workday. Simple queries: 2–5 min. Medium: 15–20 min. Complex: 60–90 min.

The AI Problem

15–70%

Hallucination rate in probabilistic AI systems for financial queries (Atil et al., 2025). Generic AI tools cannot safely replace analyst judgment on financial data.

The DetQ Answer

0%

Hallucination rate by architecture. Deterministic guard rules enforce what the user explicitly said, never what the model inferred. Same input, same output, every time.

02 Architecture

AI for extraction.
Code for everything else.

01

— Guard Rules
G-01
Numeric Intent Gate

Numeric filters only apply if the user explicitly stated a number, unit, or comparison word. Never inferred.

G-02
Entity Guard

Issuer and ticker filters only kept if a company name or ticker was explicitly in the query.

G-03
Stop Token Protection

TLAC, GREEN, M&A, UOP never treated as tickers. Mapped to deal flag fields instead.

G-04
Ranking Pre-Extraction

Sort intent extracted before AI runs, preventing ranking language from becoming filter criteria.

G-05
UOP Trigger Lock

Use-of-proceeds bucket only activates on "UOP" or "use of proceeds", never from green or TLAC.

G-06 through G-10
Demand, Exclusion, Pricing, Rating Agency, Sector Sanity

Five additional guards enforcing explicit language requirements for specialist filters.
